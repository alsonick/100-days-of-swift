## Day 1 - Topics Summary

- Variables ✅

  - Learned about how variables work in the [Swift Programming Language](https://www.swift.org/).
  - Results: 6/6

- Strings and integers ✅

  - Covered the [String](https://developer.apple.com/documentation/swift/string) and [Int](https://developer.apple.com/documentation/swift/int) data types.
  - Results: 6/6

- Multi-line strings ✅

  - In Swift you can create multi-line strings using the `"""` syntax.
  - The `"""` triple quotes _must_ start and end on a new line.
  - If you don't want the new line break in your multi-line string then add a `\` backslash on the end of each line.
  - Results: 12/12

- Doubles and booleans ✅

  - Covered the [Double](https://developer.apple.com/documentation/swift/double) and [Bool](https://developer.apple.com/documentation/swift/bool) (Boolean) data types.
  - Doubles are numbers with decimal points.
  - Booleans are simply `true` or `false`.
  - Results: 12/12

- String interpolation ✅

  - Covered how to use string interpolation work in Swift.
  - The `\()` is the syntax for string interpolation.
  - Example: `var str = "You are \(age) years old."`
  - Results: 6/6

- Constants ✅

  - In Swift you can declare a constant using the `let` keyword.
  - You can't change the value with constants.
  - It's best practice to always use constants unless you don't need to.
  - Results: 6/6

- Type annotations ✅

  - In Swift you can mark your variables or constants with type annotations.
  - By default Swift uses [type inference](https://en.wikipedia.org/wiki/Type_inference) to infer the data type of variables and constants.
  - Results: 6/6

- Simple types: Summary ✅
  - Results: 6/6
