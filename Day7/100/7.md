## Day 7 - Topics Summary

- Using closures as parameters when they accept parameters ✅

  - Closures can also accept parameters.
  - Results: 10/12

- Using closures as parameters when they return values ✅

  - Closures can also return values.
  - Results: 11/12

- Shorthand parameter names ✅

  - If a closure only accepts a single value then you can exclude the return type.
  - If a closure only returns a single value then Swift can infer the return type and we can remove the `return` keyword.
  - Results: 6/6

- Closures with multiple parameters ✅

  - Closures can also accept multiple parameters.
  - Results: 12/12

- Returning closures from functions ✅

  - In Swift functions can also return closures.
  - You need to mark the closures's after the `->` symbol.
  - Results: 12/12

- Capturing values ✅

  - If you use any external values inside your closure, Swift captures them - stores them alongside the closure, so they can be modified even if they don't exist any more.
  - Results: 11/12
