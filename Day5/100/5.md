## Day 5 - Topics Summary

- Writing functions ✅

  - In Swift you can use functions to re-use code.
  - Swift functions start with the `func` keyword.
  - Results: 11/12

- Accepting parameters ✅

  - Functions in swift can accept parameters.
  - Results: 10/12

- Returning values ✅

  - Functions can also return data.
  - You must use the `return` keyword to return a value.
  - You can return multiple types using a tuple.
  - Results: 12/12

- Parameter labels ✅

  - Swift allows us to provide two name for each parameter.
  - The first name is the external purposes and the second one is for internal purposes.
  - Results: 12/12

- Omitting parameter labels ✅

  - You can omit the external parameter by using the `_` underscore syntax.
  - Results: 12/12

- Default parameters ✅

  - You can give your own parameters a default value just by writing an `=` sign after its type followed by the default you want to give it.
  - Results: 12/12

- Variadic functions ✅

  - Variadic functions can accept any number of parameters.
  - The `print()` function uis variadic.
  - Results: 6/6

- Writing throwing functions ✅

  - Functions can fail sometimes if they have bad input or if something went wrong internally.
  - Swift lets us throw error from functions by marking them as `throws` before their return type.
  - Results: 12/12

- Running throwing functions ✅

  - Functions that can throw errors must be wrapped inside the `do` code block.
  - You must use the `try` keyword before every function that might throw an error.
  - The `catch` block lets you handle errors gracefully.
  - Results: 6/6

- inout parameters ✅

  - Parameters passed into a Swift function are `constants` by default.
  - If you want to change the function's parameter values then you must mark then as `inout` before the type.
  - Changes are reflected outside the function.
  - Results: 6/6

- Functions: Summary ✅
  - Results: 12/12
